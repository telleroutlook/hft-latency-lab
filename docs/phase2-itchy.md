## Week 4–5: 受控优化，逐发验证

按"收益/风险"排序逐项试，每项必须先有计数器证据：

### 2.3.1 数据布局 / cache 行重排（通常最稳收益）
- [ ] 热字段塞同一 cache 行，冷热分离

### 2.3.2 分支消除
- [ ] 基于 2.2 的 branch-miss 数据
